============================================
Save a MongoDB Connection
============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can save a new MongoDB connection from the Relational Migrator
:guilabel:`Connections` page, or when creating or modifying a sync job.

Before you Begin
----------------

To save a connection to a MongoDB Atlas cluster, you must be :ref:`logged in <rm-atlas-log-in>` with Atlas.

Steps
-----

.. procedure::

   .. step:: Add a new connection.
      
      Under :guilabel:`MongoDB`, click :guilabel:`Add connection`.

   .. step:: Enter your connection information.

      If you are using MongoDB Atlas, you can select your Atlas cluster 
      from the :guilabel:`Select a cluster` list, or you can enter the
      :manual:`MongoDB connection string <mongodb-uri>`.

      If you are using a self-hosted deployment, you must enter the MongoDB connection string.

   .. step:: Optionally, in the :guilabel:`Database` field, enter the name of 
      the database to connect to.

   .. step:: Enter the :guilabel:`Username` and :guilabel:`Password` to use 
      for authentication.
         
      Checking :guilabel:`Save password` saves your password securely on your 
      machine. Relational Migrator automatically fills in the 
      :guilabel:`Password` field on future uses.

   .. step:: Specify a :guilabel:`Connection name` and optionally an 
      :guilabel:`Environment tag`.
         
      Environment tags color code and organize connections for convenience.
      Leaving the drop-down empty is the same as selecting the "None" tag.

   .. step:: Optionally, click :guilabel:`Test connection`.

      Relational Migrator attempts to authenticate and connect to the
      database.
         
   .. step:: Click :guilabel:`Save`.
