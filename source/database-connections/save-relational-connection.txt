============================================
Save a Relational Database Connection
============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can save a new relational database connection from the Relational Migrator
:guilabel:`Connections` page, or when creating or modifying a sync job.


Steps
-----

.. procedure::
   :style:  normal

   .. step:: Add a connection.
      
      Under :guilabel:`Relational` click :guilabel:`Add connection`.

   .. step:: In the :guilabel:`Database type` drop-down, select the database type and input its information.

      .. tabs::

         .. tab:: Oracle
            :tabid: db-oracle

            To input a :ref:`connection string <rm-connection-strings>` directly:
   
            a. Above the :guilabel:`JDBC URI` field, enable the
               :guilabel:`Enter URI manually` toggle.

            #. Paste your connection string into the :guilabel:`JDBC URI`
               field.

            Or, create a connection string by entering the following:

            .. include:: /includes/table-oracle-connection-fields.rst

         .. tab:: SQL Server
            :tabid: db-sql

            To input a :ref:`connection string <rm-connection-strings>` directly:
   
            a. Above the :guilabel:`JDBC URI` field, enable the
               :guilabel:`Enter URI manually` toggle.

            #. Paste your connection string into the :guilabel:`JDBC URI`
               field.

            Or, create a connection string by entering the following:

            .. include:: /includes/table-sql-connection-fields.rst

         .. tab:: MySQL
            :tabid: db-mysql

            To input a :ref:`connection string <rm-connection-strings>` directly:
   
            a. Above the :guilabel:`JDBC URI` field, enable the
               :guilabel:`Enter URI manually` toggle.

            #. Paste your connection string into the :guilabel:`JDBC URI`

.. step:: Enter the :guilabel:`Username` and :guilabel:`Password` to use for authentication.
      
   Checking :guilabel:`Save password` saves the password securely on 
   your machine. Relational Migrator automatically fills in the 
   :guilabel:`Password` field for the current project.

.. step:: Optionally, if you are connecting to a SQL, MySQL, or PostgreSQL server, you can connect using SSL:
                     
   a. Toggle from :guilabel:`General` to :guilabel:`SSL`.
                     
   #. Select an SSL mode.

.. step:: Specify a :guilabel:`Connection name` and optionally an :guilabel:`Environment tag`.
      
   Environment tags color code and organize connections for 
   convenience. Leaving the drop-down empty is the same as selecting 
   the "None" tag.

.. step:: Optionally, click :guilabel:`Test connection`.

   Relational Migrator attempts to authenticates and connects to the
   database.
      
.. step:: Click :guilabel:`Save`.
